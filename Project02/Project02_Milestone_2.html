Project02 — Milestone 2: Controlled Sensor Modification (Small Changes)

Timing: Weeks 4–6
Tools: Raspberry Pi + GrovePi+
Development Tool: RealVNC editor or VS Code SSH (keep changes small)
Submission: OneNote Artifact (PDF) + modified .py file(s) to D2L

Objective

Make safe, small modifications to a provided sensor script and prove your changes work.

Student Tasks

  1. Choose ONE sensor script to modify.
  2. Make TWO small modifications:
    • threshold message, clearer prints, faster/slower delay, trigger counter, etc.
  3. Run and verify changes with output evidence.
  4. AI check (required): ask what bug to watch for; verify against it.

VS Code SSH Exercise (Required)

Task 5 — Use VS Code SSH to edit on the Raspberry Pi

  1. In VS Code, install the Remote - SSH extension (if not already installed).
  2. Connect to your assigned Raspberry Pi using SSH (profile provided in class).
  3. Open the folder that contains your chosen sensor script.
  4. Save your modified script as a NEW file:
    • milestone2_<sensorname>.py

Task 6 — Run from the VS Code terminal (on the Pi)

In the VS Code terminal connected to the Pi, run:

      python3 milestone2_<sensorname>.py

Task 7 — Troubleshooting (required)

Create one safe troubleshooting scenario and fix it:

  • Example: stop the program with Ctrl+C, then restart it cleanly
  • Example: introduce a small typo, observe error, fix it

Add screenshots of the error and the fix to your OneNote Artifact.

OneNote Artifact Requirements

  • Before/After code snippet
  • Screenshot showing new behavior
  • AI prompt + summary
  • Reflection: which change was hardest and why
